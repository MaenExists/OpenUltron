# Memory Layout

OpenUltron stores experience in plain markdown so both humans and the model can read it directly.

Folders:
- `experiences/`: loop entries grouped by day.
- `knowledge/`: distilled knowledge notes.
- `skills/`: reusable playbooks or procedures.
- `summaries/`: periodic rollups.
- `actions/`: execution logs for approved actions.

Queue:
- `actions_queue.md`: pending/proposed actions for approval.

State:
- `state.md`: current loop status and counters.
